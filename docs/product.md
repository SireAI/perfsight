# PerfSight Product Positioning

## 定位

`PerfSight` 是一个面向 Android 测试场景的、**无侵入、进程外、可长时间后台运行**的内存泄漏监控工具。

它不要求应用接入 SDK，也不要求改业务代码。  
核心目标不是交互式分析，而是：

- 在 `monkey`、压测、稳定性回归这类长时间场景里持续监控
- 在疑似泄漏出现时主动保留现场
- 为后续归因提供可直接使用的 `HPROF` 和元信息

## 核心价值

- **无侵入**
  不改 App 代码，不引入额外运行时逻辑，不影响业务接入成本。

- **进程外监控**
  基于 `adb`、`dumpsys meminfo` 和 `/proc`，从进程外观察应用状态，避免把监控逻辑带进应用内部。

- **低扰动、适合长时间运行**
  适合后台持续跑，尤其适合 `monkey`、压测、稳定性回归等长时间场景。

- **主动发现问题**
  不依赖人工盯盘，而是基于持续增长、回落不足、`Activities / ViewRootImpl` 等规则自动判定疑似泄漏。

- **主动保留现场**
  在合适时机自动或手动触发 `HPROF dump`，并把文件拉回本地，保留归因证据。

## 适用场景

- `monkey` 随机测试
- 长时间压测
- 稳定性回归
- 无人值守后台监控
- 疑似泄漏后的自动现场保留

## 和 Android Studio Profiler 的区别

`Android Studio Profiler` 更适合：

- 开发阶段
- 人工在线连接
- 交互式操作
- 短时间精细分析

`PerfSight` 更适合：

- 测试阶段
- 长时间后台运行
- 无人值守监控
- 自动发现疑似泄漏
- 自动保留 dump 现场

一句话概括：

**PerfSight 不是一个交互式分析器，而是一个面向长时间测试场景的泄漏哨兵。**
